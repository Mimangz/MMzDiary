                      1，文件

                      在Java中，有File对象表示一个文件信息。

                      我们先来看一个例子：

                      // 实例化一个文件对象，用文件路径初始化
                      File file = new File("E:\\中转\\a.png");
                      // 打印文件名
                      System.out.println(file.getName());
                      // 打印该文件的绝对路径
                      System.out.println(file.getAbsolutePath());
                      // 打印该文件的所在目录
                      System.out.println(file.getParent());
                      // 是否为一个文件
                      System.out.println(file.isFile());
                      // 是否为一个文件夹
                      System.out.println(file.isDirectory());

                      File类有以下常用方法：

                      • getName 获取文件名
                      • getAbsolutePath 获取绝对路径
                      • getParent 获取所在文件夹路径
                      • isFile 判断其是否是文件
                      • isDirectory 判断其是否是文件夹
                      • exists 判断该文件是否存在
                      • delete 删除该文件

                      File的构造函数中是以文件路径作为初始化，可以是相对路径或者是绝对路径，还可以指定一个不存在的路径然后用createNewFile创建该空文件，也可以指定一个不存在的目录路径然后用mkdir方法创建目录或者mkdirs直接创建多级目录。

                      如果指定了一个目录，还可以获得该目录下的所有文件列表：

                      • list 列出该目录下所有文件名，返回字符串数组
                      • listFiles 列出该目录下所有文件，返回File数组

                      文件基本操作还是很简单的，大家尝试一下就会了。

                      1. 读取文本文件

                      我们以一个读取文本文件的代码为例。先新建文本文档如下：

                      然后用代码读取：

                      xxxxxxxxxx
                      // 实例化一个文件对象
                      File file = new File("E:\\中转\\新建文本文档.txt");
                      // 实例化文件流，构造函数传入文件对象
                      InputStream inputStream = new FileInputStream(file);
                      // 实例化流读取器，构造函数传入文件流
                      Reader fileStreamReader = new InputStreamReader(inputStream);
                      // 实例化缓冲读取器以读取内容
                      BufferedReader bufferedReader = new BufferedReader(fileStreamReader);
                      // 读取全部行
                      String line;
                      while ((line = bufferedReader.readLine()) != null) {
                          System.out.println(line);
                      }
                      // 最后记得关闭读取器，否则会占用内存
                      bufferedReader.close();
                      fileStreamReader.close();
                      inputStream.close();

                      看起来有点复杂，但其实并不难。我们这里用到了以下类：

                      • File 文件信息，不必多说

                      • FileInputStream 文件输入流，构造函数是指定要读取为流的文件

                      • InputStreamReader 输入流读取器，构造函数指定要读取哪个流

                      • BufferedReader 缓冲读取器，构造函数指定流读取器，其中：

                        • readLine读取一行，然后指针会往下移动一行。每执行一次该方法，就会往下读取一行，返回值就是该行内容

                      通常我们读取文本文件就是这样的。

                      2. 写入文本文件

                      上面读取使用输入流，那么写入当然就是输出流了。还是以下面为例：

                      xxxxxxxxxx
                      // 实例化一个文件对象
                      File file = new File("E:\\中转\\新建文本文档1.txt");
                      if (!file.exists()) {
                          file.createNewFile();
                      }
                      // 实例化文件输出流
                      OutputStream fileOutputStream = new FileOutputStream(file);
                      // 实例化输出流写入器
                      Writer outputWriter = new OutputStreamWriter(fileOutputStream);
                      // 实例化缓冲写入器
                      BufferedWriter bufferedWriter = new BufferedWriter(outputWriter);
                      // 写入文件
                      bufferedWriter.write("111");
                      // 换行
                      bufferedWriter.newLine();
                      // 再写一行
                      bufferedWriter.write("222");
                      // 最后记得关闭写入器和流，否则文件内容不会保存
                      bufferedWriter.close();
                      outputWriter.close();
                      fileOutputStream.close();

                      可见其实和上面读取很类似，只是把上面的输入流变为输出流、读取器变成写入器了。

                      有时由于编码问题可能产生乱码，因此在实例化InputStreamReader或者OutputStreamWriter时还可以指定编码：

                      xxxxxxxxxx
                      // 实例化输出流写入器，指定为UTF-8编码
                      Writer outputWriter = new OutputStreamWriter(fileOutputStream, "UTF-8");
                      // 实例化流读取器，并设定编码为UTF-8
                      Reader fileStreamReader = new InputStreamReader(inputStream, "UTF-8");

                      编码通常和要读取或者写入的文件一致即可，常用的就是UTF-8或者GBK。

                      (2) 字节流

                      可见刚刚字符流是操作一些文本文件，但是对于二进制文件，我们就要用字节流了。不过在学习字节流之前，我们最好是先认识一下计算机中文件的存储形式。

                      1. 计算机中文件的储存形式

                      之前我们就学习到了：文件在计算机中实质上是以二进制形式存储的，我们常常用kb，mb等等作为文件的大小单位。

                      在大学计算机基础中我们就学过：计算机存储容量的基本单位是字节(byte)，而每个0或1就是一个位(bit)，位是内存的最小单位。1字节由8个位组成，这也是为什么我们byte类型的数据表示范围只能从-128 ~ 127。大家先要弄清楚字节和位的概念和关系。

                      也就是说，计算机一次最少也得存入8个0和1组成的数据，并且读取时一次也最少得读取8个0和1组成的数据，计算机的读写都是八位八位地进行操作的，每个位就是0或者1。

                      不过和我们Java中数据类型不同的是：计算机中的字节存储形式都是无符号的，因此对应的，在计算机中一个字节的表示范围是0 ~ 255，转换成二进制就是00000000 ~ 11111111。

                      无符号的意思就是没有负值的数据，只表示0以及大于0的数。

                      我们有时候需要读取一个文件的原始数据内容，也就是这个文件的二进制内容，如果说里面每一个字节都显示成8位的二进制数据例如00000000或者01100100这样的，不仅看得难受，还难以分析。

                      而如果把这些字节数据都转换成十六进制，就简洁多了。0x00刚好可以表示字节00000000，而0xff刚好可以表示字节11111111。这也是为什么计算机中十六进制也被广泛使用。

                      大家如果安装了VSCode编辑器的话，可以在里面安装hexdump for VSCode这个插件，这个插件可以读取文件的二进制内容。

                      如下就是一个png图片的二进制数据：

                      所以说，计算机储存的文件实质上都是0和1，也就是一些二进制的数据，也就是说，其实每个文件的实质可以说就是一堆byte类型的数据。

                      2. 二进制文件和文本文件

                      事实上，在计算机中所有的文件应当称作是二进制文件，因为文件的储存形式都是二进制的。但是人们通常还是会把文件分为文本文件和二进制文件这两种。

                      • 文本文件：能够被记事本或者文本编辑器打开并显示的文件，例如我们的txt格式的文本文档，以及源代码文件等等。文本文件无论其扩展名是什么，都可以修改成txt打开或者是直接用文本文档打开，也就是说，文本文件中所有的字节都可以通过已有的字符集映射为人类可读的字符
                      • 二进制文件：不能够被记事本或者文本编辑器打开读取的文件，如果说你用记事本或者文本编辑器强行打开，你只会看到很多无法阅读的乱码（即使你切换很多字符集去读取它），也就是说，二进制文件中存在一些甚至是大部分字节都不能够通过已有的字符集映射为人类可读的字符，我们常用的版本管理工具Git，就是通过读取并判断文件中是否存在字节值为00000000（控制字符\0）的字节来判断这个文件是文本文件还是二进制文件，若存在这样的字节就是二进制文件

                      3. 读取二进制文件

                      我们来看一个简单示例：

                      xxxxxxxxxx
                      // 实例化一个文件对象
                      File file = new File("E:\\中转\\a.png");
                      // 初始化文件流
                      InputStream inputStream = new FileInputStream(file);
                      // 新建一个字节型数组，存放读取的文件字节内容，因此字节数组长度要和文件大小一致，File的length方法就是获取文件大小的，单位是字节(b)
                      byte[] content = new byte[(int) file.length()];
                      // 读取文件为字节流并将读取到的内容存入字节数组content中
                      inputStream.read(content);
                      // 关闭流
                      inputStream.close();
                      // 看看字节流中的内容
                      for (byte b : content) {
                          System.out.print(b + " ");
                      }

                      这样，就读取了一个二进制文件的内容了！

                      可见我们把文件中每个字节都读取出来并输出了，不过Java中的字节型是有符号的，而计算机中表示的字节型是无符号的，因此实际上需要做一个无符号字节型和有符号字节型的转换，大家可以自行上网查阅。

                      除此之外，FileInputStream还可以使用缓冲区的方式读取文件：

                      xxxxxxxxxx
                      // 实例化一个文件对象
                      File file = new File("E:\\中转\\a.png");
                      // 初始化文件流
                      InputStream inputStream = new FileInputStream(file);
                      // 新建一个字节型数组作为缓冲区，缓冲区大小为1024
                      byte[] buffer = new byte[1024];
                      // 打印每一次读取的数据的长度
                      int length = 0;
                      while ((length = inputStream.read(buffer)) != -1) {
                          System.out.println(length);
                      }
                      inputStream.close();

                      可见其read方法返回了这一次读取的长度，参数就是缓冲区，每一次读取的数据都会放在这个作为缓冲区的字节数组中，也就是说每执行一次read方法，就会往下读取缓冲区长度的数据量并储存在缓冲区中，并覆盖原有在缓冲区中的内容（上一次读取的内容）。根据代码也可以看出：读取到末尾之后，没有内容可以读了，就会返回-1，因此可以用这个方法判断是否读取完毕。

                      大家多加尝试就可以理解了。

                      4. 写入二进制文件

                      其实方法也很类似：

                      xxxxxxxxxx
                      // 实例化一个文件对象
                      File file = new File("E:\\中转\\bb.bin");
                      if (!file.exists()) {
                          file.createNewFile();
                      }
                      // 随便定义一些字节内容
                      byte[] bytes = {1, 2, 3, 4, 5};
                      // 初始化文件输出流
                      OutputStream outputStream = new FileOutputStream(file);
                      // 写入文件，参数为写入的字节内容，为字节数组
                      outputStream.write(bytes);
                      // 关闭流，否则文件不会保存
                      outputStream.close();

                      我们使用二进制浏览器看看写入的文件：

                      好了，学习了二进制文件的读写，我们是不是也可以轻而易举地写一个复制文件的方法了呢？只需要读取被复制文件为字节流再写入至目标位置即可：

                      xxxxxxxxxx
                      /**
                       * 复制文件，把文件从原文件路径复制到目标文件路径，目标文件路径目录不存在会自动创建
                       *
                       * @param origin      原文件路径
                       * @param destination 目标文件路径
                       */
                      public static void copyFile(String origin, String destination) throws Exception {
                          File originFile = new File(origin);
                          File destFile = new File(destination);
                          File parentDirectory = new File(destFile.getParent());
                          if (!parentDirectory.exists()) {
                              parentDirectory.mkdirs();
                          }
                          if (!destFile.exists()) {
                              destFile.createNewFile();
                          }
                          // 实例化输入输出流，输入流读取被复制文件，输出流写入文件
                          InputStream is = new FileInputStream(originFile);
                          OutputStream os = new FileOutputStream(destFile);
                          int lengthEachRead;
                          // 读取缓冲区
                          byte[] buffer = new byte[4096];
                          while ((lengthEachRead = is.read(buffer)) != -1) {
                              os.write(buffer, 0, lengthEachRead);
                          }
                          is.close();
                          os.close();
                      }

                      这里提一下FileOutputStream的write，除了上述例子中一个参数的用法，还有这里三个参数用法，分别是：

                      • 写入的字节内容，传入字节数组
                      • 写入偏移，通常写0就行
                      • 写入长度，一般为写入的内容长度

                      可见这里write是一次次地写入的，每次写入指定长度而非全部一次性写入：

                      可见每写入一次指针就向后移动，上述写入偏移参数就是相对于上图指针而言的，也就是从偏移指针后的多少个字节开始写入。可见上述复制文件的方法是每次读取4096个字节，然后就将其写入目标文件。

                      5. 对象的序列化和反序列化

                      其实我们还可以把Java中的对象变成流，甚至写入到文件中。

                      首先创建一个类，保证其使用了Serializable接口：

                      ​x
                      import java.io.Serializable;
                      

                      /**
                       * 猫类
                       */
                      public class Cat implements Serializable {
                      

                          /**
                           * 猫的名字
                           */
                          private String name;
                      

                          /**
                           * 猫的年龄
                           */
                          private int age;
                      

                          public String getName() {
                              return name;
                          }
                      

                          public void setName(String name) {
                              this.name = name;
                          }
                      

                          public int getAge() {
                              return age;
                          }
                      

                          public void setAge(int age) {
                              this.age = age;
                          }
                      

                      }

                      然后创建一个对象，可以将其可以序列化为流：

                      xxxxxxxxxx
                      // 实例化猫
                      Cat cat = new Cat();
                      cat.setName("大橘");
                      cat.setAge(2);
                      // 实例化字节序列输出流
                      ByteArrayOutputStream bos = new ByteArrayOutputStream();
                      // 实例化对象写入流
                      ObjectOutputStream oos = new ObjectOutputStream(bos);
                      // 序列化（读取）猫对象，并存到上述字节序列流中
                      oos.writeObject(cat);
                      // 输出至文件
                      OutputStream fileOutput = new FileOutputStream(new File("E:\\中转\\cat.bin"));
                      fileOutput.write(bos.toByteArray());
                      // 最后关闭全部
                      fileOutput.close();
                      oos.close();
                      bos.close();

                      当然，也可以反序列化：

                      xxxxxxxxxx
                      // 读取文件
                      File file = new File("E:\\中转\\cat.bin");
                      byte[] content = new byte[(int) file.length()];

                      6. 字符串和字节流

                      到此，我们学习了字节流和字符流，可见这两种流中内容是不一样的，字节流中的内容是二进制数据，当然在Java中这些二进制数据会表示成byte形式，字符流中的内容就全是字符了，在Java中字符流中是char数据。

                      在计算机中，事实上可以被文本文档打开的文件，也就是文本文件，其本质上也是二进制形式存储的，我们在计算机看到的任何文字，其在硬盘和内存中事实上都是通过二进制字节形式保存，只不过被计算机通过字符集转换成了我们可读的字符。在前面我们就已经学习到了字符集这个概念，更多关于字符集的知识大家可以上网查阅资料。

                      因此，字符串事实上也可以转换成字节数据，通过字符串对象的getBytes方法即可：

                      xxxxxxxxxx
                      String s = "abcde";
                      byte[] b = s.getBytes();
                      for (byte bb : b) {
                          System.out.print(bb + " ");
                      }

                      也可以指定编码进行转换：

                      xxxxxxxxxx
                      // 转换成字符对应的UTF-8编码对应的字节形式
                      byte[] b = s.getBytes(StandardCharsets.UTF_8);

                      反过来，如果有一个byte数据，也可以转换成字符串，利用字符串的构造函数即可：

                      xxxxxxxxxx
                      byte[] bytes = {97, 98, 99};
                      // 传入byte数组到String的构造函数即可将其转为字符串
                      String s = new String(bytes);
                      System.out.println(s);

                      当然，也可以指定编码进行转换：

                      xxxxxxxxxx
                      // 传入byte数组到String的构造函数即可将其转为字符串，以UTF-8编码转换
                      String s = new String(bytes, StandardCharsets.UTF_8);
